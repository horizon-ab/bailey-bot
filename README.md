# Bailey Bot

## Functionality

### Monitoring

Bailey Bot monitors all messages sent in specified channels. Specified channels can be set by moderators.  Only the first few messages (specified by `message_threshold`) sent by a given user will be checked.

### Judgement

Bailey Bot uses an LLM (DeepSeek-R1-Distill-Llama-8B from HuggingFace) to review the message content and will return a judgement score. If the judgement score is above a certain threshold, it will automatically delete the message and ban the offending user. If the judgement score is above a lower threshold, it will ping a moderator for manual review. If testing mode is enabled, only manual review will be permitted. Both of these thresholds can be set by a moderator.

## Commands

`/test`
Enables/disables testing mode. In testing mode the bot is not allowed to enforce judgement (i.e. ban users).

`/set`
Adds a channel to be monitored.

`/remove`
Removes a channel to be monitored.

`/review`

`/confidence_auto`
Sets confidence threshold for auto ban. If `score >= confidence_auto`, then the bot will automatically enforce judgement. By default, this is `0.9`.

`/confidence_manual`
Sets confidence threshold for manual review. If `confidence_auto > score >= confidence_manual`, then the bot will ping a moderator for manual review. By default, this is `0.7`

`/message_threshold`
Sets message threshold for monitoring. The first `message_threshold` messages will be checked by the bot. 

## Model

The `model` folder includes everything you need to retrain and fine-tune the model. Inside, the `data` folder includes the .csv file found [here](https://www.kaggle.com/datasets/shibe123/discord-phishing-scam?resource=download) plus some extra synthetic data generated by Gemini, appended at the very end of the file. The `config` folder should include configuration files for running tests, but this currently is not implemented. The `outputs` folder includes the results of all training runs. Unfortunately their performances are not documented so you will have to test them yourselves in `inference.py`, which provides the functions that bailey-bot calls along with a simple performance test which can be run with `python inference.py` (assuming you have the dependencies installed of course). Just remember to change the `RUN_FOLDER` to test the results of a specific training run. The `src` folder contains `model.py` which defines the structure of the model (distilbert plus a few linear layers and a sigmoid layer), `dataset.py` which loads the data in `data`, and `main.py` which actually trains the model. As a side note, the texts are parsed with special symbols such as `<USER>` or `<URL>` to improve the quality of the model.

### Training

Note that these instructions are written for macOS. I pray that you are knowledgeable enough to execute the following instructions with Windows or Linux.

1. First verify that you are in the `model` folder. This is where all subsequent commands should be run.
2. If you want to edit the hyperparameters, edit `src/main.py`. You can change the epochs, training rate, etc.
3. Create a virtual environment with `python -m venv .model-venv`, run the environment with `source .model-venv/bin/activate`, and install dependencies with `pip install -r model-requirements.txt`
4. Run the training script with `python src/main.py`. This took about 4-5 minutes on my M1 Pro Mac. 
5. If you want to test the results of the new model, first find the latest run folder by running `ls outputs`(note that the format is YYYYMMDD-HHMMSS) and copy the name of that folder. Go into `inference.py` and change `RUN_FOLDER` with the name of the copied folder. Finally run `python inference.py`. The model's evaluations will be displayed under each text.
